## 2. 添加新策略

### 方式一：手动添加

1. 点击页面右上角 **「➕ 添加策略」**
2. 选择 **「手动添加」**
3. 填写表单：
   - **玩法名称**：给策略起个名字（如"暗图困难"）
   - **地图**：选择所属地图（暗图、深空等）
   - **平均时长**：单次通关时长，格式 `MM:SS`（如 `02:30`）
   - **难度**：简单 / 中等 / 困难 / 专家
   - **DPS值 / 生存值**：可选，填写数字
   - **标签**：可选，如"多多"、"速刷"等
4. 点击 **「添加策略」**

### 方式二：截图识别（推荐）

1. 点击 **「➕ 添加策略」**
2. 选择 **「添加截图」**
3. 拖拽或选择游戏内「玩法详情弹窗」截图
4. 点击 **「🔍 解析预览」**
5. 系统自动识别并填入：
   - 成本物品（消耗品名称 + 数量）
   - 核心掉落（物品名称 + 数量 + 稀有度）
6. 编辑玩法名称、地图、时长、难度、标签
7. 点击 **「✅ 直接添加」**

> [!TIP]
> 截图识别需要截图包含完整的玩法详情弹窗，物品名称和数量都要清晰可见。

## 4. 筛选和排序

### 筛选

在策略列表上方有三个下拉框：

| 筛选器 | 说明 |
|--------|------|
| 全部地图 | 只显示指定地图的策略 |
| 全部难度 | 只显示指定难度的策略 |
| 全部标签 | 只显示包含指定标签的策略 |

三个筛选器同时生效（AND 逻辑）。

### 排序

点击表格列标题可排序：

| 列标题 | 说明 |
|--------|------|
| 成本 | 按成本火价升序/降序 |
| 掉落 | 按掉落总价值排序 |
| 净收益/时 | 按每小时净收益排序 |
| ROI | 按投资回报率排序 |

## 5. 推荐卡片

推荐卡片位于首页中部，展示 ROI 排名前3的策略：

### 卡片结构

```
┌──────────────────────────────┐
│  🥇  无天赋        暗图 困难  │  ← 排名 + 名称 + 地图/难度徽章
│  ⏱ 00:16  [多多]            │  ← 时长 + 标签
├──────────────────────────────┤
│  DPS 111,111  生存 222,222   │  ← DPS/生存属性
├──────────────────────────────┤
│        净收益/时              │
│      +6503.6 🔥               │  ← 大字突出显示（正数绿色）
│  掉落 28.90🔥  成本 --🔥    │  ← 副属性
├──────────────────────────────┤
│  [图][图][图]                │  ← 加点图缩略图
└──────────────────────────────┘
```

### 指标说明

| 指标 | 说明 |
|------|------|
| 净收益/时 | 每小时净赚的火价（掉落 - 成本） |
| 掉落 | 核心掉落总价值（按当前火价计算） |
| 成本 | 成本物品总消耗（按当前火价计算） |

## 6. 外网访问

需要从外部网络（不在同一局域网）访问时，使用 Cloudflare 隧道：

### Windows

1. 双击运行 `start_tunnel.bat`
2. 等待出现 `https://xxxx.trycloudflare.com`
3. 分享该链接即可外网访问

### macOS / Linux

```bash
cloudflared tunnel --url http://localhost:19878
```

> [!WARNING]
> Cloudflare 临时隧道（无账号）不保证稳定性，仅适合临时测试使用。

## 数据说明

### 火价数据

- TL_GS 依赖 TL_item_monitor 数据库中的火价记录
- 启动时自动读取最新火价
- 每5分钟自动刷新
- 如火价显示 `--`，表示未连接到数据库

### 收益计算

```
掉落总价值 = Σ(每件物品数量 × 该物品火价)
成本总消耗 = Σ(每件消耗品数量 × 该物品火价)
净收益     = 掉落总价值 - 成本总消耗
净收益/时  = 净收益 ÷ 平均时长(分钟) × 60
ROI (%)    = (掉落总价值 ÷ 成本总消耗 - 1) × 100
```

### 数据存储

- 策略数据保存在 `data/tl_gs.db`（SQLite）
- 截图文件保存在 `static/uploads/`
- 所有数据存储在本地，不会上传到任何服务器
